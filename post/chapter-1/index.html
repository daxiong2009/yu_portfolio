Biomechanics Data: Wide to Long

January 12, 2021

This short article discuss how to turn “Wide” data into “Long” data. “Wide” data is very common in biomechanics studies. Investigators often measure many different variables(e.g., knee flexion/extension, COM, GRF…), those variables are arranged in column. “Wide” data format is easy to understand, but difficult to be handled if you when to plot the data. So, you will need the “Long” data.

Experimental scenario: I am interested in examine how the two types of footwear affect the the lower limb coordination during running at 3 different speed: slow, normal and fast. The method I estimate the lower limb coordination is based on modified vector coding method, which is created by Chang and colleagues(2008). Generally, this method categorize the coupling angle into 4 distinct regions that indicate 4 different lower limb coordination pattern. The detailed algorithm can be found in their article:[Quantifying rearfoot-forefoot coordination in human walking, DOI: 10.1016/j.jbiomech.2008.07.024].


1. First, load pandas and numpy package

import pandas as pd
import numpy as np

2. Then, load the data

  • wear: footwear, 2 conditions: sd vs. bf
  • 4 distinct regions of coordination pattern (stubnames)
    • in: In-Phase
    • anti: Anti-Phase
    • knee: Knee-Phase
    • ankle: Ankle-Phase
  • 3 running speeds (separate with stubnames by “_")
    • s: slow
    • n: normal
    • f: fast

3. Next, we are about to reshape the wide dataframe

  • In the raw data, each of three running speeds will have 4 same coordination regions. We will first need to create a separate variable of speed, which is utilized to index what coordination regions corresponds to which speed.
  • step 1: add additional identifier (subject_id) to the raw data
id=np.arange(1,21)
subject_id=np.concatenate([id,id])
dat['subject_id']=subject_id
  • step 2: wide_to_long
    • dat: name of dataframe
    • stubnames: this should be self-explained
    • i: this is the identifier, can be a single variable or a list
    • j: this is the new variable you need to create
    • sep: specify the symbol that separate with the stubnames
    • suffix: usually you can copy this line
dat=pd.wide_to_long(dat,
                   stubnames=['in','anti','knee','ankle'],
                   i=['wear','subject_id'],
                   j='speed',
                   sep='_',
                   suffix=r'\w+')

  • step 3: reset row index

    • Looking at the image above, you may think everything looks fine. But actually NOT!
    • wide_to_long function is powerful, but its downside is that it usually screw up the row index. If you don’t fix this, you will not be able to proceed to the next step.
dat=dat.reset_index()
dat

  • step 4: use melt function to further reshape the dataframe
    • melt() is not as powerful as wide_to_long (). But melt () is relatively easy to apply, and which is suitable to use in this case, since I already used wide_to_long().
    • id_vars: identifier variables
    • melt() does not need to specify the name of new variable after reshape, it will automatically create a “value” and “variable”. If you are not satisfy the name, you can change the column names later
dat=pd.melt(dat,
            id_vars=['wear','subject_id','speed'],
            value_vars=['in','anti','knee','ankle'])

That’s it! Although pandas have other powerful tools to convert wide data to long data, my personal experience is that method I discussed above is sufficient to solve most of the problem
